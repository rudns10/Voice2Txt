# Voice2Txt 작업 이력 (요청 → 결과)

이 대화에서 요청한 내용과 처리 결과를 시간순으로 정리한 기록.

---

## 1. 단계별 빌드 (개발프롬프트 W0~W5)

| 단계 | 요청 | 결과 |
|------|------|------|
| **W0** | 개발프롬프트대로 개발 시작(골격) | 솔루션 + Core/Cli/Gui 3계층, WinUI 3 빈 창, DI 구성. (.NET 9, WindowsAppSDK 1.7, unpackaged self-contained) |
| **W1** | 녹음 | NAudio WASAPI → 16kHz mono WAV, 대기/녹음중 화면, 타이머·파형·일시정지/정지 |
| **W2** | 목록 + 저장 + 메타데이터 | SQLite 저장, 저장 다이얼로그, 사이드바 목록 + 검색, 재시작 유지 |
| **W3** | 변환 엔진(MVP) | whisper.cpp(BLAS) 외부 프로세스 호출, 변환중 화면, 진행률, 모델 자동 다운로드. (공식 Vulkan 프리빌트 없어 BLAS로 MVP) |
| **W4** | 결과 + 재생 + 내보내기 | 결과 화면, MediaPlayer 재생, 구간 클릭 재생, TXT/SRT 내보내기, 복사/편집/삭제 |
| **W5** | Whisper.net 정식 통합 | Whisper.net 1.9 (Vulkan→CPU), DI 한 줄로 엔진 교체. Vulkan 로드 확인, 속도 비교 |

## 2. UI를 목업에 맞춤
- "UI가 목업과 다르다" → 목업 HTML(5화면) 기준으로 **전면 재구성**
- 적용: 라이트/다크 테마 팔레트, **빨강 녹음 버튼**, **세그먼트 토글**(일반/음성→텍스트), 모노폰트 타임코드, 사이드바 스타일, 결과 화면 플레이어/transcript, optbox 스타일

## 3. 기능 추가 / 수정 요청

| 요청 | 결과 |
|------|------|
| 파형이 목소리에 반응하게(파도처럼 출렁) | 진행성 사인파 애니메이션 + **피크 계산 버그 수정**(WaveFormatExtensible 미인식) |
| 모델 small/medium 어떻게 고르나 | **모델 선택 UI** 추가(설정 저장), 이후 **사이드바 하단으로 이동**(어디서든 표시) |
| "텍스트로 변환" 버튼이 안 눌림 | 빈 transcript 리스트가 버튼을 덮던 버그 수정 + **변환을 백그라운드 실행**(UI 비차단) + **실패 시 에러 다이얼로그** |
| 목록 우클릭 메뉴 | **이름 수정 / 삭제** 컨텍스트 메뉴 |
| 창이 작음 | 기본 크기 확대(1440×920) + **창 크기/위치 기억**(재시작 복원) |
| 결과 화면에서 메인 복귀 | 사이드바 **"+ 새 녹음"** 버튼 |
| 스페이스바 단축키 | 대기=시작 / 녹음중=**일시정지·재개** / 결과=재생·일시정지 (입력칸 포커스 시 일반 입력) |
| 기존 오디오 파일 변환 | **파일 불러오기**(mp3/m4a/wav 등) → 16kHz 변환 → 자동 STT |
| "새 녹음" 후 이전 메시지 잔류 | 상태 메시지 초기화 |
| 빈 목록/검색결과 안내 | placeholder 문구 추가 |
| 제목줄 다듬기 | 마이크 아이콘 + "Voice2Txt" 커스텀 제목줄 + Mica 배경 |

## 4. 배포 / 문서
- **배포 패키징**: self-contained 게시 → zip ([scripts/package.ps1](scripts/package.ps1)). 개발환경 없는 PC에서 exe 더블클릭 실행
- **모델 동봉**: 앱이 앱폴더 모델도 인식 + 스크립트 `-Models` 옵션. `with-small` zip은 인터넷 없이 즉시 변환
- **문서화**: [README.md](README.md)(사용법/빌드/배포), [CLAUDE.md](CLAUDE.md)(구조/아키텍처/관례)

## 5. 질문(개념 설명)으로 답한 것
- 일반/음성→텍스트 모드 차이 → (현재 음성모드는 자막 미리보기 자리만, 실시간 자막은 범위 밖) → **그대로 유지**
- 복사 버튼 역할 → transcript 전체를 클립보드로 복사
- Whisper small/medium 등 모델 크기 의미
- W6(로컬 REST 서버)가 무엇인지 → 사내 연동용, **미구현(후순위)**
- 모델 동봉의 의미

## 6. 2차 개선 세션 (기능 · UX · 성능 · 배포)

### 6-1. 기능
| 요청 | 결과 |
|------|------|
| 일괄 삭제(하나씩 지우기 귀찮음) | **선택 모드**(항목 체크박스) + **[전체 선택] 토글** + **[삭제 (n)]**. 확인창 1회, Escape 오삭제 방지 |
| 파일을 목록에 끌어다 놓기 | 사이드바 **드래그&드롭 가져오기**(wav·mp3·m4a·aac·wma·flac, 여러 개 동시) — 버튼/드롭 공용 `ImportFileAsync` |
| 가져오면 바로 변환되지 말게 | 가져오기 후 **자동 변환 제거** — 목록에만 추가, 변환은 사용자가 직접 |
| 같은 이름이면 (1),(2) 증가 | 표시 이름도 유니크 파일명 따르게(회의 → 회의 (1) → 회의 (2)…) |
| 실시간 자막 못 찾고 헤맴 | 모델 드롭다운에서 분리 → **독립 "실시간 자막" 토글**(설정 저장). medium/large도 실시간 사용 가능 |

### 6-2. 편집 / 다이얼로그
| 요청 | 결과 |
|------|------|
| 편집 취소 버튼 | 편집 모드에 **[편집 취소]**(원문 복귀) 추가 |
| 편집 저장 후 내용이 안 보이고 00:00로 합쳐짐 | 편집을 "한 줄=한 구간"으로, 저장 시 **원래 타임스탬프와 재매핑** → 타임스탬프 목록으로 복귀 (WinUI TextBox `\r` 줄바꿈 처리) |
| 삭제·취소 버튼 위치 | **삭제=오른쪽 / 취소=왼쪽**(Secondary 배치, Escape는 취소로) |

### 6-3. 성능 / 정확도 (무음 환각 · GPU 가속)
| 요청 | 결과 |
|------|------|
| 무음에 "감사합니다 / 자막 제공 / 광고 포함" 등 환각 | ① **구간 음량(RMS) 무음 게이트** → 소리 없으면 공백 ② 유튜브 정형구 필터(포함 매칭). 임계값 80으로 **조용한 실제 발화는 보존** |
| CUDA 가속 부재 지적 | **CUDA 런타임 추가**, 로드 순서 **CUDA→Vulkan→CPU** 자동 폴백. 인텔=Vulkan, NVIDIA=CUDA (이 노트북 폴백 실측 확인) |
| large가 medium보다 빠른 이유 | 앱의 large는 **large-v3-turbo**(디코더 4층 vs medium 24층) → 더 빠르고 정확. 카탈로그에 turbo 추가 |

### 6-4. 디자인
- **모던 SaaS 리디자인**: 인디고 액센트(라이트/다크), 전역 라운드(컨트롤 8/카드 12), 부드러운 표면
- 녹음 버튼 **부드러운 코랄 레드**(눈부심 완화), "새 녹음" 인디고 강조 버튼
- 변환 화면 라벨에서 "· 로컬(오프라인) 처리" 문구 제거

### 6-5. 배포 / 저장소
- 배포 패키지 기본값을 **모델 미포함(none)** 으로 — 첫 변환 때 다운로드. 폐쇄망은 `-Models`로 동봉
- self-contained **exe 배포본 생성**(dist zip, ~390MB, CUDA 포함)
- Git **원격 2곳**: GitHub(`origin`) + 사내 Gitea(`gitea`). `git push` 한 번에 **양쪽 동시 푸시**하도록 설정

### 6-6. 질문(개념)으로 답한 것
- Whisper 동작(멜 스펙트로그램 → 인코더 → 디코더), **어텐션 층**, 왜 무음에 환각이 생기나
- **OpenAI vs Anthropic(Claude)**, CUDA vs Vulkan, GPU가 CPU보다 빠른 이유
- 모델 크기별 차이·속도(small/medium/turbo **실측**), turbo vs 정통 large-v3
- 모델 **다운로드 시점**·인터넷 필요 여부, **모델 동봉** 장단점
